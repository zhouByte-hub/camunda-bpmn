# 场景：企业级「服务器采购多级审批工作流」
## 核心定位
- 多角色：采购申请人、项目组长、部门经理、CEO、财务
- 多平台：PC 管理后台、移动端审批、邮件通知系统、财务系统

## 完整流程链路
1. **发起采购申请**

    采购人员提交服务器采购申请，填写采购金额、配置需求等信息，流程自动进入审批环节。


2. **项目组长审批**

    项目组长对采购需求的合理性进行初步审核，可通过或驳回申请。驳回则流程直接终止。


3. **部门经理审批**

    部门经理审核采购必要性与预算合规性。审批不通过则退回项目组长重新审议；审批通过后按采购金额分流。


4. **金额分流判断**

    系统自动判断采购金额：金额 ≥ 2000 元流转至 CEO 审批；金额 < 2000 元直接进入财务审批。


5. **CEO 审批（大额采购）**

    大额采购（≥ 2000 元）须经 CEO 审批。CEO 驳回则退回项目组长重新审议；CEO 通过后流转财务审批。


6. **财务审批**

    财务对预算来源、费用科目进行审核。审批通过后触发拨款购买；审批不通过则发送驳回通知邮件。


7. **拨款购买**

    财务审批通过后，系统自动调用服务接口完成拨款，执行服务器采购操作。


8. **邮件通知**

    流程结束时，自动发送邮件通知相关人员审批结果（通过或驳回）。


9. **超时逾期处置**

    各审批节点（项目组长、部门经理、财务、CEO）均设有超时定时器。审批逾期后自动触发逾期处置服务，发送逾期提醒通知并结束流程。


10. **撤销消息处理**

    流程支持外部消息事件（`task_revoke`）触发撤销子流程，自动发送逾期邮件通知相关负责人。

## 适配 Camunda8 核心能力
- 多角色 User Task、条件网关（金额分流、审批结果路由）
- 非中断边界定时器事件（各节点超时逾期处置）
- 消息触发事件子流程（撤销通知）
- 服务任务（拨款购买、逾期通知、邮件发送）
- Task Listener（审批日志记录）
